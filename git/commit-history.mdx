ted's Commit History panel shows a visual timeline of your repository's commits with author, date, message, and diff details. Instead of switching to a terminal and running `git log`, you get a fully interactive graph where hovering any commit reveals its full change summary — including a file-level breakdown of what was inserted or deleted.

## Opening the panel

Click the **clock (history) icon** in the left sidebar to bring up the Commit History panel. Alternatively, look for the **Commit History** tab inside the Terminal panel area at the bottom of the editor.

## Reading the commit graph

The graph lists commits from newest to oldest. Each row shows:

| Column | Content |
|---|---|
| **Hash** | Short commit hash (first 7 characters) |
| **Message** | Commit message summary line |
| **Author** | The committer's name |
| **Date** | Commit timestamp |

Commits are connected by graph lines that visualize branching and merging in your repository's history. The panel loads the most recent 100 commits by default.

## Inspecting commit details

Hover over any row in the commit graph to reveal a **detail popup** for that commit. The popup shows:

- **Author** and full **date/time** of the commit
- The complete **commit message**
- A diff summary: **files changed**, **lines inserted (+)**, and **lines deleted (−)**
- The short **commit hash**

Move the mouse away from the row to dismiss the popup.

## File churn

Beyond individual commits, ted tracks **file churn** — the number of times each file has changed within a configurable time window (for example, the last 30 days).

- High-churn files are those being modified most frequently.
- The built-in AI agent reads churn data to prioritize context and suggestions for files that are actively evolving.
- You can view churn metrics inside the Commit History panel to get a quick sense of which parts of the codebase are hottest right now.

<Tip>
  Use commit history together with churn data to understand *how* a specific file has evolved and *how often* it changes — a powerful combination for onboarding to an unfamiliar codebase.
</Tip>
