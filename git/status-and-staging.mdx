The Source Control panel gives you a live view of your repository's working tree status. As you edit files, ted continuously reads the git index and surfaces every changed, added, or deleted file — no manual refresh needed. From the same panel, you stage your changes, write a commit message, and commit, all without leaving the editor.

## Opening the panel

Click the **branch icon** in the left sidebar to open the Source Control panel. The panel opens docked to the sidebar and stays in sync with your working tree as you edit.

## File status indicators

Each file in the Source Control panel carries a short status badge that tells you what changed.

| Badge | Meaning |
|---|---|
| **M** | Modified — a tracked file with uncommitted changes |
| **N** | New — a previously untracked file that has been added |
| **D** | Deleted — a tracked file that has been removed |

Files appear in two sections: **Staged Changes** (files queued for the next commit) and **Changes** (files modified but not yet staged).

## Staging and unstaging files

<Steps>
  <Step title="Stage a file">
    Hover over a file in the **Changes** section and click the **`+`** icon that appears to its right. The file moves to the **Staged Changes** section.
  </Step>
  <Step title="Unstage a file">
    Hover over a file in the **Staged Changes** section and click the **`−`** icon. The file is removed from the staging area and returns to **Changes**.
  </Step>
</Steps>

## Writing a commit message and committing

<Steps>
  <Step title="Write your message">
    Click the commit message input field in the Source Control panel and type your commit message. ted does not enforce any format, but conventional commit messages (e.g., `feat: add login flow`) work well.
  </Step>
  <Step title="Commit staged changes">
    Click the **Commit** button to create the commit. ted commits exactly the files listed in the **Staged Changes** section.
  </Step>
</Steps>

<Note>
  You must have at least one file staged and a non-empty commit message before the **Commit** button becomes active.
</Note>

## Viewing a diff

Click any modified file in the Source Control panel to open a **diff view** in the editor. ted renders a comparison using its built-in diff viewer, with removed lines in red and added lines in green.

## Git gutter

While you edit, ted marks changed lines directly in the editor's left margin — no need to open the Source Control panel to see what's new.

| Marker color | Meaning |
|---|---|
| Green bar | Line was added since the last commit |
| Red bar | Line was deleted since the last commit |

Gutter markers update as you edit. They reflect the difference between your current working copy and what was last committed.

<Note>
  ted currently supports local commits only. Push and pull to remote repositories are not yet available.
</Note>
