The Source Control panel gives you a live view of your repository's working tree status. As you edit files, ted continuously reads the git index and surfaces every changed, added, or deleted file â€” no manual refresh needed. From the same panel, you stage your changes, write a commit message, and commit, all without leaving the editor.

## Opening the panel

Click the **branch icon** in the left sidebar to open the Source Control panel. You can also reach it through the Command Palette:

1. Press `Ctrl+Shift+P` / `Cmd+Shift+P` to open the Command Palette.
2. Type **Source Control** and press `Enter`.

The panel opens docked to the sidebar and stays in sync with your working tree as you edit.

## File status indicators

Each file in the Source Control panel carries a short status badge that tells you what changed.

| Badge | Meaning |
|---|---|
| **M** | Modified â€” a tracked file with uncommitted changes |
| **A** / **new** | Added â€” a new, previously untracked file |
| **D** | Deleted â€” a tracked file that has been removed |

Files appear in two sections: **Staged Changes** (files queued for the next commit) and **Changes** (files modified but not yet staged).

## Staging and unstaging files

<Steps>
  <Step title="Stage a file">
    Hover over a file in the **Changes** section and click the **`+`** icon that appears to its right. The file moves to the **Staged Changes** section.
  </Step>
  <Step title="Unstage a file">
    Hover over a file in the **Staged Changes** section and click the **`âˆ’`** icon. The file returns to **Changes** (git resets it to HEAD).

    To unstage everything at once, click **Unstage All** at the top of the Staged Changes section.
  </Step>
</Steps>

<Tip>
  You can stage or unstage all files in one click using the **Stage All** (`+` on the Changes section header) and **Unstage All** controls â€” useful when you want to start a commit from a clean slate.
</Tip>

## Writing a commit message and committing

<Steps>
  <Step title="Write your message">
    Click the commit message input field at the top of the Source Control panel and type your commit message. ted does not enforce any format, but conventional commit messages (e.g., `feat: add login flow`) work well.
  </Step>
  <Step title="Commit staged changes">
    Click the **âœ“ checkmark** button, or press `Ctrl+Enter` / `Cmd+Enter`, to create the commit. ted commits exactly the files in the **Staged Changes** section.
  </Step>
</Steps>

<Note>
  You must have at least one file staged and a non-empty commit message before the commit button becomes active.
</Note>

## Viewing a diff

Click any modified file in the Source Control panel to open a **diff view** in the editor. ted renders a side-by-side comparison using its built-in merge/diff viewer, with the original version on the left and your working copy on the right. Removed lines appear in red and added lines in green.

## Git gutter

While you edit, ted marks changed lines directly in the editor's left margin â€” no need to open the Source Control panel to see what's new.

| Marker color | Meaning |
|---|---|
| Green bar | Line was added since the last commit |
| Orange / yellow bar | Line was modified since the last commit |

Gutter markers update in real time as you type. Click a marker to see a compact inline diff of just that hunk.

<Note>
  ted currently supports local commits only. Push and pull to remote repositories will be available in a future release.
</Note>
