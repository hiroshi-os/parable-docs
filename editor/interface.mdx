ted arranges its workspace into three primary regions: a collapsible sidebar on the left, the editor area in the center, and an optional terminal panel that slides up from the bottom. Every region is designed to stay out of your way until you need it — the frameless window and minimal chrome keep your code front and center at all times.

## Sidebar

The sidebar is a panel area docked to the left edge of the window. A **NavTape** strip at the top of the sidebar lets you scroll between the available panels by clicking the chevron buttons, scrolling your mouse wheel, or clicking a panel label directly. The active panel's content fills the area below.

Toggle the sidebar open or closed with <kbd>Ctrl+B</kbd> / <kbd>Cmd+B</kbd>, or click the sidebar icon in the titlebar.

## Editor Area

The editor area occupies the main body of the window and consists of two sub-components: the **tab bar** at the top and the **editor pane** below it.

### Tab Bar

The tab bar is the horizontal strip that lists every open file as a tab. Each tab shows the filename and a dirty-state indicator (a dot **•**) when the file has unsaved changes. Scroll your mouse wheel over the tab bar to reveal tabs that extend beyond the visible area, and middle-click any tab to close it instantly.

### Editor Pane

The editor itself is powered by [CodeMirror 6](https://codemirror.net/), a highly extensible browser-based editing engine. Out of the box you get:

- **Syntax highlighting** — driven by web-tree-sitter grammars running in a background Web Worker for supported languages, and by CodeMirror's built-in lezer grammars for the rest.
- **Indentation guides** — subtle vertical lines that mark indentation levels inside your code.
- **Line numbers** — displayed in the left margin; toggleable in **Settings → Editor → Line Numbers**.
- **Code folding** — collapse and expand blocks using the fold markers in the gutter.
- **Bracket matching** — the matching bracket is highlighted when your cursor sits next to one.
- **Git gutter** — a thin column between the line numbers and your code that shows `+` (added) and `~` (modified) markers for lines that differ from the last git commit.
- **Git blame** — an inline ghost text annotation on the current line that shows the commit author and date.

## Terminal Panel

The integrated terminal gives you a full PTY session (bash on macOS/Linux, PowerShell on Windows) without switching to a separate app.

<Steps>
  <Step title="Toggle the terminal">
    Press <kbd>Ctrl+J</kbd> (Windows/Linux) or <kbd>Cmd+J</kbd> (macOS) to show or hide the terminal panel.
  </Step>
  <Step title="Open a new terminal session">
    Press <kbd>Ctrl+Shift+`</kbd> to open an additional terminal session alongside any existing ones.
  </Step>
  <Step title="Resize the panel">
    Drag the horizontal divider between the editor and the terminal to resize. ted remembers the height you set between sessions.
  </Step>
  <Step title="Run commands">
    The terminal starts in your workspace root, so you can run build scripts, git commands, or any CLI tool immediately.
  </Step>
</Steps>

<Tip>
  You can have the terminal open and still interact with the editor above it — focus switches automatically when you click in either region.
</Tip>

## Command Palette

The Command Palette is the fastest way to trigger editor actions or navigate your project without reaching for the mouse.

Press <kbd>Ctrl+Shift+P</kbd> / <kbd>Cmd+Shift+P</kbd> to open it. As you type, ted performs fuzzy matching against all available commands — file operations, view toggles, settings, and actions contributed by installed extensions.

<Info>
  The Command Palette closes automatically when you select a result or press <kbd>Escape</kbd>.
</Info>

## Settings

Open the Settings popup with <kbd>Ctrl+,</kbd> / <kbd>Cmd+,</kbd> from anywhere in the editor. Settings are organized into four categories — **General**, **Appearance**, **Editor**, and **Files** — and support both user-level defaults and per-project overrides in the same popup.

## Swarms View

The Swarms view is a separate, full-window perspective designed for **multi-agent coordination**. Instead of a single agent working in the Agent panel, Swarms lets you monitor and direct several agents operating in parallel — each with its own task, output stream, and file context.

Switch to the Swarms view by clicking the **swarms** tab in the titlebar. Switch back to the editor view the same way. The editor, terminal, and sidebar remain exactly as you left them when you return.

<Note>
  The Swarms view is an advanced feature intended for orchestrating complex, multi-step workflows. For everyday single-agent assistance, the Agent sidebar panel is sufficient.
</Note>
