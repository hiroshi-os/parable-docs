ted keeps every file you open as a tab in the tab bar — the horizontal strip running across the top of the editor area. You can have as many tabs open as you like, scroll through them with your mouse wheel, and run entirely separate workspace sessions in the same window. All of this is designed so you can stay focused on your code rather than managing windows.

## Opening Files

There are three quick ways to open a file:

- **File Explorer** — click any file in the sidebar Explorer panel. The file opens in a new tab to the right of the currently active tab.
- **Command Palette** — press <kbd>Ctrl+Shift+P</kbd> / <kbd>Cmd+Shift+P</kbd>, type `open file`, and select the result. You can also use <kbd>Ctrl+O</kbd> / <kbd>Cmd+O</kbd> to open a file dialog directly.
- **New file** — press <kbd>Ctrl+N</kbd> / <kbd>Cmd+N</kbd> to create a blank untitled file in the current workspace.

## Tab Types

Not every tab holds a text file. ted supports four distinct tab types, each rendered differently in the tab bar:

<CardGroup cols={2}>
  <Card title="Editor" icon="file-code">
    A standard CodeMirror editing surface for a source file. This is the default type when you open any file from the explorer or command palette.
  </Card>
  <Card title="Diff" icon="code-compare">
    A side-by-side diff view (powered by `@codemirror/merge`) for comparing two versions of a file. Opened automatically when you review a git change.
  </Card>
  <Card title="Browser" icon="globe">
    An in-app web browser tab for previewing local dev servers or documentation without leaving ted. Open one with `browser: open new tab` in the Command Palette.
  </Card>
  <Card title="Extensions" icon="puzzle-piece">
    The Extensions Marketplace tab, where you can browse and install extensions that add new languages, tools, or integrations to ted.
  </Card>
</CardGroup>

## Scrolling Through Tabs

If you have more tabs open than fit in the visible tab bar, scroll your mouse wheel horizontally over the bar to reveal the ones off-screen. The active tab always receives focus when you click it.

## Closing Tabs

- **Mouse** — click the **×** icon on the tab, or middle-click the tab directly.
- **Keyboard** — press <kbd>Ctrl+W</kbd> / <kbd>Cmd+W</kbd> to close the currently active tab.

<Warning>
  If the file has unsaved changes (indicated by the **•** dot on the tab label), ted will prompt you to save, discard, or cancel before closing. You will not lose work silently.
</Warning>

## Dirty State Indicator

A small dot (**•**) appears on the tab label whenever a file has changes that have not been written to disk. The dot disappears as soon as the file is saved — either by pressing <kbd>Ctrl+S</kbd> / <kbd>Cmd+S</kbd> or, if Auto Save is enabled, automatically as you type.

## Navigating Between Tabs

Use <kbd>Ctrl+Tab</kbd> / <kbd>Ctrl+Shift+Tab</kbd> to cycle forward and backward through your open tabs without lifting your hands from the keyboard. You can also jump directly to a specific tab with <kbd>Ctrl+1</kbd> through <kbd>Ctrl+9</kbd>.

## Workspaces

ted supports multiple **workspace sessions** running concurrently within the same window. Each workspace is fully independent — it has its own open folder, its own set of tabs, and its own terminal session.

The **titlebar** shows all open workspaces as labeled tabs. Click a workspace tab to switch to it instantly, or click the **+** button to create a new workspace. Close a workspace with the **×** on its titlebar tab.

<Info>
  Workspace state (open tabs, scroll positions, active file) is persisted between restarts, so you pick up exactly where you left off.
</Info>

## Keyboard Shortcuts

| Action | Windows / Linux | macOS |
|---|---|---|
| Command Palette | `Ctrl+Shift+P` | `Cmd+Shift+P` |
| Open file dialog | `Ctrl+O` | `Cmd+O` |
| New file | `Ctrl+N` | `Cmd+N` |
| Save file | `Ctrl+S` | `Cmd+S` |
| Close active tab | `Ctrl+W` | `Cmd+W` |
| Next tab | `Ctrl+Tab` | `Ctrl+Tab` |
| Previous tab | `Ctrl+Shift+Tab` | `Ctrl+Shift+Tab` |
| Jump to tab 1–9 | `Ctrl+1` – `Ctrl+9` | `Ctrl+1` – `Ctrl+9` |
