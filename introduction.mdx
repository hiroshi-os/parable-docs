**ted** is a minimalist desktop code editor built for developers who want AI agents as a first-class part of their workflow. Instead of bolting AI features onto an existing editor, ted is designed around the agent loop from the start — letting you run, inspect, and iterate on AI agents alongside your code, git history, and terminal in a single native application.

## What is ted?

ted is a native desktop application built on [Tauri 2](https://tauri.app), which means it runs as a real desktop app on macOS, Linux, and Windows — not in a browser, not on a server. The editor surface is powered by [CodeMirror 6](https://codemirror.net), giving you a fast, extensible editing experience with [tree-sitter](https://tree-sitter.github.io/tree-sitter/) syntax highlighting for a broad set of languages.

The project is currently in active development at version **0.4.0-alpha**. Core features are functional, but the API surface and configuration format may change between releases as the project matures.

## What ted is not

ted is a **native desktop application**. It is not:

- A **cloud IDE** — ted runs entirely on your local machine. There is no server, no account, and no cloud sync.
- A **web app** — ted is not accessed through a browser. It is packaged as a native binary via Tauri.
- A **server process** — ted does not run in the background or expose a port. It opens as a standard desktop window.

If you need a browser-based or cloud-hosted editor, ted is not the right tool. If you want a fast, local, agent-first editor that runs like any other desktop app, ted is built for you.
