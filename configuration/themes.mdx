ted includes four built-in themes that affect both syntax highlighting and the editor chrome — every color in the interface, from the sidebar background to keyword tokens in your code, is defined by the active theme. Switching themes is instant and applies everywhere without restarting.

## Switching Themes

Open settings with `Ctrl+,` on Windows/Linux or `Cmd+,` on macOS, navigate to the **Appearance** section, and select a theme from the **Theme** dropdown. The change takes effect immediately.

You can also set the theme directly in your settings file:

```json
{
  "theme": "ted solarized light"
}
```

## How the Theme System Works

ted uses a semantic theming approach. Each theme defines two sets of tokens:

- **Syntax tokens** — colors for language constructs like keywords, strings, comments, function names, and type annotations. These power CodeMirror's syntax highlighting layer.
- **UI tokens** — colors for chrome elements like the sidebar, tab bar, status bar, panel backgrounds, borders, and icon tints.

Both sets are applied atomically when you switch themes, so there is no mismatch between editor colors and UI colors. ted does not currently support loading external or custom theme files — all four themes are bundled with the application.

## Per-project Theme Overrides

You can set a different theme for a specific project by adding a `theme` key to that project's `.ted/settings.json`:

```json
{
  "theme": "ted light"
}
```

This overrides your global theme preference only while that project folder is open. Your user-level theme is restored when you switch to another project.

## Appearance Settings

Beyond the theme, two other settings in the Appearance category affect the visual experience:

### Sound Volume

The navigation tape spinner in ted plays a subtle click sound. The `volume` setting controls its loudness on a scale of 0–100:

```json
{
  "volume": 0
}
```

Set `volume` to `0` to silence it completely. The default is `50`.

### UI Motion Blur

The `uiBlur` setting adds a subtle motion blur effect during panel transitions:

```json
{
  "uiBlur": true
}
```

It is disabled by default. Enable it if you prefer animated panel transitions with a cinematic feel, or leave it off for a snappier, lower-motion interface.

<Tip>
If you experience performance issues during panel transitions, make sure `uiBlur` is set to `false` (the default). Motion blur compositing can be GPU-intensive on lower-powered machines.
</Tip>
